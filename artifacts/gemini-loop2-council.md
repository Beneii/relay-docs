# Loop 2 Council: Highest-Impact Work — Gemini Council

1. **Integrated Onboarding & Setup Guide:** Currently, a first-time user after signup is dropped into an empty dashboard with no guidance. Building a "Quick Start" checklist that walks the user through creating their first app, sending a test notification via `curl` or the SDK, and downloading the mobile app is the single most impactful way to reduce churn in the first 5 minutes. Without a clear path to that first "buzz" on their phone, users will drop off before they even understand the value proposition.

2. **Heartbeat Monitoring (Dead-man's Switch):** This is the strongest feature for converting Free users to Pro. For vibe-coders building trading bots or autonomous agents, knowing when their system *stops* working is more critical than receiving status updates. By offering a "Heartbeat" feature that triggers a critical alert if a webhook isn't received within a set window, Relay moves from being a "nice-to-have" notification pipe to an essential piece of infrastructure that justifies the $7.99/mo subscription.

3. **relay.json Validation & UI Previews:** Since the mobile app now supports auto-fetching `relay.json` manifests, the `edit-app` UI in the dashboard must surface validation errors and show a live preview of the theme color, icon, and tabs. If a developer's manifest is broken or misconfigured, it leads to a broken mobile experience that erodes trust. Surfacing these issues in the dashboard provides a tight feedback loop for the developer and reinforces the "Mobile Runtime" story by showing them exactly how their tool will appear on a device.
