# Relay Product Roadmap

## Phase 1: Now (Foundational)
- **Multi-Platform Webhook Interface:** A functional system for saving web dashboards as native apps on mobile devices while providing unique webhook tokens for each.
- **Push Notification Delivery:** Core capability to receive HTTP POST requests from external systems and deliver real-time push notifications via Expo.
- **Unified Authentication:** A cross-platform auth system allowing seamless handoff between web and mobile via secure hash-fragment tokens.
- **Pro Subscription Model:** Integration with Stripe for monthly and annual billing to unlock higher usage limits.

## Phase 2: Next (Near-term Implementation)
- **Notification History:** A dedicated view to browse and search previously received alerts, providing a record of system events over time.
- **Metadata Events:** Support for passing and displaying rich JSON metadata within notifications to provide more context before opening a dashboard.
- **Advanced Redirect Logic:** Smarter deep-linking that can open specific views or states within a dashboard based on webhook payload parameters.
- **Refined Permissions & Sharing:** Capability to manage notification preferences per dashboard and potentially share dashboard access with team members.

## Phase 3: Future (Long-term Vision)
- **Interactive Notifications:** Actionable alerts that allow users to trigger simple webhook responses (e.g., "Acknowledge", "Restart Server") directly from the lock screen.
- **Status Monitoring Dashboards:** Built-in "health checks" that track the uptime and performance of the monitored systems themselves within Relay.
- **Native Widget Support:** Home screen widgets for iOS and Android to provide at-a-glance status updates without opening the app.
- **Third-Party Integrations Library:** First-class connectors for popular tools like GitHub, Grafana, and Home Assistant to simplify setup for non-technical users.
